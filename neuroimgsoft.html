1. FSL (Ubuntu)

You will need administrative privileges in order to install FSL.

1.1. Update all your current packages

  • Open your Terminal application:
    • Click on the nine dots in the left bottom of the screen. It will show a menu with all the installed applications.
    • Scroll down until you see an application called Terminal and click on it. The Terminal will open.
    • Navigate to the folder where you downloaded the FSL installer. For example, if the file was downloaded in the Downloads directory, type $ cd ~/Downloads/
  • Type $ sudo apt-get autoclean to clean the cache by removing files that cannot longer be downloaded from the local repository.
  • Type $ sudo apt-get autoremove in order to remove libraries that were previously depencies of some programs but are no longer being used (to clear space).
  • (recommended) Type $ sudo apt-get update to update the package list with the latest versions of your programs (does not actually upgrade or install those new versions).
  • (recommended) Type $ sudo apt-get upgrade to upgrade those programs (already installed in your machine) that have new versions available and install the new versions.
  • Type Y when prompted for confirmation
  • Type $ sudo apt-get dist-upgrade. Because $ sudo apt-get upgrade only upgrades packages that are already installed in your machine, whenever a new version of a program requires other packages that are not yet installed, $ sudo apt-get upgrade will not update those packages. However, in those cases $ sudo apt-get dist-upgrade will still perform the upgrade and resolve any dependencies. If you get any errors or warnings of packages that were kept back with $ sudo apt-get upgrade, use $ sudo apt-get dist-upgrade instead.
  • Type Y when prompted for confirmation

1.2. Check your Python installation

Make sure that you have Python 2 installed. To do that, type:
$ python --version
If you have Python 2 installed, you should see something like this (the exact version will vary but should start with a 2):
Python 2.7.18
If Python 2 is not installed, you should see something like this:
-bash: python: command not found
And you can install it with the following command:
$ sudo apt install python

Don't worry about having two versions of Python in case you already have Python3. You can always use $ python2 <your_python2_script> to run your python2 scripts and $ python3 <your_python3_script> to run your python3 scripts. Or just $ python <your_python_script> to run the the python version that you have for default.

1.3. Register the new license

Register at fsldownloads_registration. Click on [FSL software license] to read and agree to the software license (it won't let you register if you don't do so). Fill the form and select your version of Ubuntu, then click Register. If you are not sure what version of Linux you have, you can type $ cat /etc/issue in the command line to find out that information.

1.4. Install the software

  • A new page will open after you register. Click on Download FSL X.X.X (X.X.X will vary depending the latest version available) to download the installer (fslinstaller.py)
  • Open your Terminal application:
    • Click on the nine dots in the left bottom of the screen. It will show a menu with all the installed applications.
    • Scroll down until you see an application called Terminal and click on it. The Terminal will open.
    • I suggest that you add the Terminal application to your favorites (By doing right click on it, then select Add to Favorites) so you can access it from the main Menu. You will use the Termal to analyze all your neuroimaging data.
    • Navigate to the folder where you downloaded the FSL installer. For example, if the file was downloaded in the Downloads directory, type $ cd ~/Downloads/
  • If you have administrative privileges:
    • If Python 2 is your default version, Type: $ python fslinstaller.py
    • If Python 3 is your default version, Type: $ python2 fslinstaller.py
    • Select /usr/local/fsl as the download location for FSL
  • Provide your password when prompted
  • Log out and back in to complete the setup. If you are connected remotely, close your ssh connection and open a new one.

1.5. Update your bash profile

  • Open your bash profile: vi ~/.bashrc
  • Paste the following lines:
    FSLDIR=/usr/local/fsl
    . ${FSLDIR}/etc/fslconf/fsl.sh
    PATH=${FSLDIR}/bin:${PATH}
    export FSLDIR PATH

1.6. Check the installation

  • Start a new Terminal session
  • Check that your environment is correct by typing: $ echo $FSLDIR. It should show the location you selected during installation.
  • Check that your path is correct by typing: $ flirt -version. The output should be: FLIRT version X.X (X.X will vary depending on the version you installed)

2.3. Potential errors during installation which are not solved in the AFNI page

  • Dependency problems:
    When installing AFNI in a new linux machine with Ubuntu 20.04.1, I got some ERROR messages when executing the command $ rPkgsInstall -pkgs ALL (step 2 from the Rinstallation section).
    The exact error messages were:
    ERROR: dependencies 'lme4', 'gamm4' are not available for package 'projpred'
    * removing '/home/monica/R/projpred'
    ERROR: dependencies 'lme4', 'pbkrtest', 'lmerTest', 'car' are not available for package 'afex'
    * removing '/home/monica/R/afex'
    ERROR: dependencies 'rstan', 'threejs' are not available for package 'shinystan'
    * removing '/home/monica/R/shinystan'
    ERROR: dependencies 'lme4', 'Hmisc' are not available for package 'arm'
    * removing '/home/monica/R/arm'
    ERROR: dependencies 'lme4', 'rstan', 'shinystan' are not available for package 'rstanarm'
    * removing '/home/monica/R/rstanarm'
    ERROR: dependencies 'rstan', 'shinystan', 'projpred', 'bridgesampling', 'nleqslv' are not available for package 'brms'
    * removing '/home/monica/R/brms'

     This is how I solved the problem:
    • Install the missing dependencies:
      $ sudo apt install r-cran-lme4 r-cran-gamm4 r-cran-projpred r-cran-pbkrtest r-cran-lmertest r-cran-car r-cran-afex r-cran-rstan r-cran-threejs r-cran-shinystan r-cran-hmisc r-cran-arm r-cran-rstan r-cran-rstanarm r-cran-brms
      You could try installing one by one if you run into more dependencies issues. Add any other dependecies that you need if you get additional errors.
    • Re-run the $ rPkgsInstall -pkgs ALL. There should be no more errors or warnings. Now it should execute in a few seconds since most of the R packages were already installed.

5.2. Install all the dependency packages

You could install all of the packages below in a single command but I prefer to install them separately in case one of them has a dependency that is not installed in my computer. To install all at the same time, list all the packages name separated by a space after the word install. Either way, install first g++ and gcc before installing the others.

  • $ sudo apt install g++
  • $ sudo apt install gcc
  • $ sudo apt install git
  • $ sudo apt install libboost-all-dev
  • $ sudo apt install zlib1g
  • $ sudo apt install zlib1g-dev

5.3. Download and install Qt 5.12.8

  • Download the installer
  • Navigate to the folder where you downloaded the installer (i.e. $ cd ~/Downloads/)
  • Change the permissions of the installation file so it can be executed: $ chmod 775 qt-opensource-linux-x64-5.12.8.run
  • Execute the installer: $ ./qt-opensource-linux-x64-5.12.8.run. You will need access to the computer screen (instead of a ssh connection) because this will open an installation GUI.
  • If you get a waring about the Qt account settings file missing, it's ok, it will be created once you log in or create an account. Check that the file was actually created after the installation (If you get the warning, it will show the expected path of the file).
  • Login to your Qt Account or sign up for one when prompted, and accept the service terms
  • Check your email and verify your Qt account before clicking Next
  • Follow the instructions of the installer. When prompted to select the components, select all of them
  • Write down the folder where Qr5.12.8 was installed (the information will be provided to you during installation) because you will need to remember it in the next section
  • You can close the Qt application if it launches
  • Re-run the commands from section 2.1. in case there is a new version of Qt available or of any of the dependencies installed in section 2.2.

5.4. Download the DSI Studio installer

  • Create a working directory where the installation files will be saved. For example, in your home directory (i.e. $ mkdir /home/myuser/dsistudio)
  • Navigate into the directory that you just created (i.e. $ cd /home/myuser/dsistudio)
  • Download the source code of DSI Studio from Github: $ git clone -b master git://github.com/frankyeh/DSI-Studio.git src
  • A folder called src should now appear in your current directory (to check type $ ls)
  • Create the following subdirectory inside the src folder: $ mkdir src/tipl
  • Download frankyeh-TIPL zip folder: $ xdg-open https://github.com/frankyeh/TIPL/zipball/master
  • Move the downloaded zip file to the tipl folder we just created if downloaded elsewhere
  • Navigate to that directory: $ cd src/tipl
  • Unzip the zip file: (i.e.$ unzip frankyeh-TIPL-2020_07-0-g8eb8d76.zip) The actual file name will vary depending on when/what version you download.
  • Move all the contents of the unzipped folder to the current directory (src/tipl): (i.e. $ mv frankyeh-TIPL-8eb8d76/* .)
  • Remove the zip and unzipped (which now should be empty) files: (i.e. rm -r frankyeh-TIPL-2020_07-0-g8eb8d76.zip frankyeh-TIPL-8eb8d76)

5.5. Compile

  • Go back to your DSI studio folder (i.e. $ cd /home/myuser/dsistudio)
  • Create a directory to build and compile: $ mkdir build
  • Navigate to that directory: $ cd build
  • Execute qmake, which is installed in the Qt5.12.5 directory (and I wrote above to remember its path). i.e. if my installation folder for Qt5 is /home/myuser/Qt5.12.8, then this is the command I would run to execute qmake: $ /home/myuser/Qt5.12.8/5.12.8/gcc_64/bin/qmake ../src
  • You should see something like this: Info: creating stash file /home/myuser/dsistudio/build/.qmake.stash
  • Compile: $ make
  • Wait until it finishes compiling. This could take several few minutes. If everything goes well, you should find dsi_studio under build directory (Which should be the current directory). To check type ls dsi_studio. You should get: dsi-studio. If there was an error in the installation, you will get something like this: ls: cannot access 'dsi_studio': No such file or directory. If this happens, do not continue with the installation, check the previous steps. Did you get any error running a previous step? Did you get a message of some package or dependency that needs to be installed? Any error should be fixed right away or the installation wont be completed successfully.
  • Remove unnecessary files: $ make clean

5.6. Download the atlas files from the Windows installer

  • Go back to the dsistudio directory: $ cd ..
  • Download the Windows installer from one of the following locations:
    • DSI Studio Google Drive folder
    • DSI Studio Dropbox folder
    • DSI Studio Box folder
    • DSI Studio One Drive folder
  • Move into the current directory (dsistudio) if downloaded elsewhere
  • Unzip the file: $ unzip dsi_studio_64.zip
  • $ cd dsi_studio_64
  • Remove unnecessary files (that are for Windows):
    • find . -name '*.dll' -exec rm {} \;
    • rmdir iconengines imageformats platforms styles
    • rm dsi_studio.exe vc_redist.x64.exe
    • rm ../dsi_studio_64.zip
  • Move the DSI Studio executable into the current directory: $ cp ../build/dsi_studio .

5.7. Check the installation

  • Cross your fingers and hope for the best
  • Try to run DSI Studio from the terminal: $ ./dsi_studio
  • If the installation finished successfully, the application should open

5.8. Potential errors during installation

It's not uncommon to run into issues while installing DSI Studio. I could not possibly describe all the possible problems you could run into but here are a few examples of errors I have run into and how I solved them:

N: Skipping acquire of configured file 'main/binary-i386/Packages' as repository 'https://download.mono-project.com/repo/ubuntu stable-focal InRelease' doesn't support architecture 'i386' while upgrading or updating your packages

In this example the error occured when updating the mono package. But it could happen for any other package. To solve the issue, I typed the following commands. Replace the list file for the one corresponding to the package you are having issues with.

  • Type $ /etc/apt/sources.list.d/ and look for the .list file corresponding to the problematic repository. In my case it was: mono-official-stable.list
  • Type $ sudo gedit /etc/apt/sources.list.d/mono-official-stable.list. Replace gedit by vi or nano or you favorite editor.
  • The file should contain one line that starts with the word deb and contains the https address from the error you got when updating your packages. Place [arch=amd64] between deb and the http address.
    For me, this is how the like look before the edits:
    deb https://download.mono-project.com/repo/ubuntu stable-focal main
    And this is how it looked after the edits: deb [arch=amd64] https://download.mono-project.com/repo/ubuntu stable-focal main
  • Re-run the update or upgrade command that gave the error
